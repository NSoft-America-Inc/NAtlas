## 오늘 결정한 것들

### NAtlas Phase 1 MVP 구현 (#1)

- **배경**: 전사 지식 탐색기 NAtlas 프로젝트 시동을 위해 Documents 상태 모니터링, SwarmVault 동기화 업데이트, LLMWiki 저장소 경로 설정 및 환경 진단 기능(Phase 1 MVP)을 탑재해야 함.
- **결정**: 
  - **프론트엔드**: Shadcn UI + Tailwind v4(Vite 플러그인 컴파일)를 채택하여 세련된 다크 Slate-Indigo 테마 레이아웃 및 컴포넌트(StatusBadge, LogViewer, Layout) 구성. TanStack Query를 활용해 30초 간격으로 `GET /documents` 및 10초 간격으로 `GET /swarmvault/status` 자동 갱신.
  - **백엔드**: FastAPI 사이드카(`src/python`)를 포트 `18420`에서 단독 기동. `state/manifests/*.json` 파일들을 인메모리 스캔하여 파일 MD5/SHA256 해시값과 교차 검증함으로써 인덱싱 상태(`indexed`, `modified`, `new`) 판정 구현.
  - **생명주기 및 프로세스 연동**: Electron 메인 프로세스(`src/main/index.ts`)에서 Node `child_process.spawn`을 통해 Python 프로세스를 비동기 실행하고, `before-quit` 시 `SIGTERM`을 전달하여 깔끔한 좀비 프로세스 방지 성사. uvicorn 포트 경합 예방 및 최대 3회 자동 재시작 로직 수립.
- **구현**:
  - **사이드카 생명주기 스폰 및 폴링**:
    ```typescript
    pythonProcess = spawn('python3', [pythonScript, '--port', '18420'], {
      env: { ...process.env, PYTHONUNBUFFERED: '1' }
    })
    ```
  - **SwarmVault SSE 실시간 업데이트**:
    ```python
    @router.post("/update")
    async def post_update():
        # asyncio.create_subprocess_exec을 사용하여 비동기로 CLI 수행 및 SSE 스트리밍
        proc = await asyncio.create_subprocess_exec(
            "swarmvault", "ingest", file_rel, cwd=llmwiki_root, ...
        )
    ```

## 기술 메모

- **Vite & Tailwind v4 통합**: `@tailwindcss/vite` 플러그인을 사용하여 `tailwind.config.js` 없이 CSS 내 `@import "tailwindcss";` 임포트 방식으로 리소스를 고속 로드하고 최적화 번들링 가능.
- **FastAPI 비동기 서브프로세스 권장**: 사이드카 내에서 CLI 명령(`swarmvault`)을 내릴 때 블로킹 함수인 `subprocess.run()` 대신 `asyncio.create_subprocess_exec()`를 활용하여, 요청 스레드를 블로킹하지 않고 실시간 SSE 로그를 부드럽게 스트리밍할 수 있도록 처리함.
